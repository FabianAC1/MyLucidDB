Farrago Roadmap

This document sketches the medium-term release plan for the Farrago project. It's light on details, which will be filled in once there are owners for tasks. Some of the Farrago-level tasks may require one or more subtasks at the level of the Fennel kernel. The current task list can be viewed from Sourceforge's Task Manager . The intention of defining a roadmap is to have some basis for task definition, prioritization, and sense of progress.

Developer-only Releases

Currently, the project is in "developer only" mode where it makes no sense for anyone other than a developer to even look at it. However, during this time, we'll still do regular releases just to have some consistent code snapshots out there for prospective developers to play with.

TPC-W Release

The first non-developer release should contain just enough to demonstrate a working, integrated system that can execute a reasonable subset of SQL via JDBC. In order to have clear exit criteria for this release, the TPC-W benchmark has been selected as an integration test because it
  • has a Java implementation available which matches our target architecture well (in particular, it's servlet-based, so it does not require a remote JDBC driver)
  • uses fairly simple SQL
  • as a benchmark, can be used to drive an initial tuning effort
  • as a standard, is something we'll want to get around to eventually anyway
Note that the intention is to use TPC-W as a sample application, not to get any meaningful performance numbers (if the initial numbers aren't terrible, it means something isn't actually working yet!). If something (e.g. a subquery) is the only thing that doesn't work, we'll just disable it and declare victory.

SQL92 Functionality and Performance Releases

Once the basics are dealt with, we can move on to the tricky stuff. The goal here is to move towards competitiveness and compatibility with existing systems. These will most likely be split up into multiple releases (possibly in parallel):
  • view definition and rewrite
  • Information Schema views into catalog
  • JDBC metadata support
  • advanced datatype support
  • access control
  • referential integrity
  • advanced SQL expression support (with commonly used expressions in kernel rather than Java)
  • ODBC driver and optimized JDBC client driver, both of which may bypass Java altogether on the server side for result set retrieval
  • performance tuning
  • hash join (acquire if possible)
  • merge sort (to replace btree-insertion sort above; acquire if possible)
  • storage management, backup/restore, etc.
  • blobs
  • distributed transaction support
  • Fennel persistence for MDR
  • resource balancing
  • heterogeneous queries

Diversification Releases

If the project survives through to this point, then many paths (e.g. federation, clustering, analytics, hardware acceleration, etc.) open up. However, it's a bad idea to start on any of these before the basics are in place.
